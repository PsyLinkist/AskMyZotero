# 开发日志

## 2026-04-21

### 硬筛选第一阶段落地

- 前端主页面新增左侧硬筛选 UI，接入：
  - `Date range`
  - `Author`
  - `Tag`
  - `Collection`
  - `Results`
- 移除未启用的 `Archive` 与 `Item type`
- 页面标题与顶部产品名统一改为 `AskMyZotero`
- 前端开始向 `/api/ask` 提交 `filters` 与 `top_k`

### Zotero 元数据接入

- 从 Zotero 本地 sqlite 补充提取：
  - `authors`
  - `tags`
  - `collections`
  - `year/date`
- 元数据从父条目下沉到 attachment，再下沉到 chunk metadata

### Collection 层级展开

- 增加 collection 层级路径展开逻辑
- 若文献位于 `A/B/C`，则支持匹配：
  - `A/B/C`
  - `A/B`
  - `A`
  - `B/C`
  - `B`
  - `C`

### 硬筛选后端链路

- 定义统一硬筛选结构：
  - `collections`
  - `tags`
  - `authors`
  - `date_range`
  - `top-k`
- 作者筛选实现为不区分大小写的模糊包含匹配
- tag / collection 多值筛选实现为命中任一即可
- 时间筛选实现为年份区间过滤

### 元数据库与检索链路改造

- 新增 SQLite 元数据库：
  - `metadata.db`
- 建立表：
  - `papers`
  - `chunks`
  - `paper_authors`
  - `paper_tags`
  - `paper_collections`
- `chunks` 表补充：
  - `chunk_id`
  - `paper_id`
  - `faiss_row_id`
  - `docstore_id`
- 索引加载/重建时同步构建 `metadata.db`

### 候选集优先检索

- 检索语义调整为：
  - 不填硬筛选：默认全库
  - 填了硬筛选且命中：先筛候选文献，再展开候选 chunks
  - 填了硬筛选但筛空：直接空结果，不回退全库
- 第二阶段进一步改为真正的候选子索引检索：
  - 先由元数据库得到候选 `paper_ids / chunk_ids / faiss_row_ids`
  - 再只在候选 `faiss_row_ids` 对应的向量子集上做搜索

### 调试与日志

- 增加 `[HardFilter]` 调试日志
- 日志可观察：
  - `candidate_source`
  - `candidate_papers`
  - `candidate_chunks`
  - `total_index_docs`
  - `raw_hits`
  - `kept_hits`
  - `final_hits`

## 2026-04-22

### 搜索框推荐词改为最近搜索驱动

- 主页面搜索框下方的推荐按钮从固定文案改为优先展示最近搜索
- 前端新增最近搜索本地存储，使用 `localStorage` 保存最近 5 条搜索记录
- 新搜索成功后会写入历史，并按“最近优先、去重、最多 5 条”重新渲染推荐按钮
- 当本地还没有搜索历史时，仍回退显示默认推荐词

### Collection 筛选交互提示优化

- 将 `Collection` 输入框 placeholder 从单值示例改为逗号分隔的多值示例
- 当前示例更新为 `e.g. DTRP, MMF`，明确提示支持多个 collection 联合输入

### Tag 筛选交互提示优化

- 将 `Tag` 输入框 placeholder 从单值示例改为逗号分隔的多值示例
- 当前示例更新为 `e.g. retrieval, rag`，明确提示支持多个 tag 联合输入
