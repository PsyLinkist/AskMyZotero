## 4. 结论

当前首要瓶颈不是“目标论文召回不到”，而是下面两层：

### 4.1 意图分流错误是第一瓶颈

最明显的问题是系统过度偏向 `paper_lookup`：

- 预测分布：`paper_lookup=27/40`
- 实际 `definition` 中有 `6/8` 被判成 `paper_lookup`
- 实际 `fact_qa` 中有 `6/8` 被判成 `paper_lookup`
- 实际 `survey` 中有 `4/8` 被判成 `paper_lookup`
- 实际 `comparison` 中也有 `3/8` 被判成 `paper_lookup`

这说明：

- 当前 intent classifier 对 “What is ... / How did ... / What workflow ...” 这类单篇论文内部问法区分不够
- 一旦被分到 `paper_lookup`，后续就会走 paper-list 聚合和 paper-list 回答路径，而不是 direct-answer 路径

典型例子：

- `testpdf-qa-002` 本应是 `definition`，被判成 `paper_lookup`
- `testpdf-qa-003` 本应是 `fact_qa`，被判成 `paper_lookup`
- `testpdf-qa-022` 本应是 `definition`，被判成 `paper_lookup`

### 4.2 证据页漂移是第二瓶颈

即使目标论文基本都能召回，页码命中率仍只有 `0.475`。

更关键的是：

- `target_hit_at_5 = 1.00`
- `page_hit_rate = 0.475`

这说明问题不在“找不到那篇论文”，而在：

- 找到了目标论文，但没有把证据聚焦到正确 section/page
- 聚合和证据选择会偏向主题相关页，而不是数据集标注的关键证据页

典型例子：

- `testpdf-qa-006` 目标论文 rank=1，但命中页是 `9,18`，标注页是 `2,3`
- `testpdf-qa-011` 目标论文 rank=1，但命中页是 `10,9`，标注页是 `14`
- `testpdf-qa-031` 目标论文 rank=1，但命中页是 `14,10,11,1`，标注页是 `2`
- `testpdf-qa-036` 目标论文 rank=1，但命中页是 `14,1`，标注页是 `2,3`

这说明当前 retrieval/aggregation 还有明显的“命中文章但证据落点不准”问题。

## 5. 对检索链路的定位

结合结果，当前检索链路的优先问题顺序建议是：

1. `qa_agent.py` 的 intent classification
2. `qa_agent.py` 的 query rewrite 是否把单篇细粒度问题改写成了找论文类问题
3. `aggregator.py` 的 paper 聚合与 evidence 选择
4. section/page 级排序信号是否过弱，导致正确论文内的错误页占优

可以暂时不把重点放在：

- 向量召回“是否完全无效”
- 纯 prompt 微调
- 全量索引重建

因为从这轮基线看：

- 目标论文/目标 chunk 大多数时候已经在前排
- 真正掉分主要发生在 intent routing 和 evidence focusing

## 6. 下一步建议

下一轮检索调优建议优先做：

1. 压低 `paper_lookup` 的误判率，先把 `definition / fact_qa / survey` 拉回 direct-answer 路径
2. 为 direct-answer 增强页级/section 级 evidence rerank
3. 为 paper-list 问题单独约束“命中文章后优先保留 abstract / intro / method 中最像 identification evidence 的页”
4. 新增评测指标：
   `intent_accuracy`、`target_hit@k`、`page_hit_rate` 继续保留，后续再对比优化前后变化
